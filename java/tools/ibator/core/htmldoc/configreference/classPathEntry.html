The <classPathEntry> Element

The <classPathEntry> element is used to add classpath locations to the classpath of the iBATOR run. The <classPathEntry> element is an option child element of the <ibatorConfiguration> element. iBATOR loads classes in several instances:

  • When loading the JDBC driver for database introspection
  • When loading custom JavaModelGenerator, JavaTypeResolvers, etc.
  • When loading root classes in the JavaModelGenerator to check for overridden methods

This element is option and is not required if you setup the classpath externally to iBATOR (e.g. with the -cp argument of the java command

Required Attributes

Attribute Description
location The full path name of a JAR/ZIP file to add to the classpath, or a directory to add to the classpath.

Optional Attributes

None

Child Elements

None

Example

This element specifies the location of a DB2 JDBC driver:

<classPathEntry location="/Program Files/IBM/SQLLIB/java/db2java.zip" />
