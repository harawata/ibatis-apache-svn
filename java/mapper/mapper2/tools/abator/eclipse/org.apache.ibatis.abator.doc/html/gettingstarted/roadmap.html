Abator Roadmap

The following are enhancements that we would like to make to Abator:

  • A better configuration file editor for Eclipse
  • A better file association mapping for Eclipse (will be accomplished in the Eclipse 3.1 version)
  • Ability to provide custom generator objects in the Eclipse environment (will be accomplished in the Eclipse 3.1 version). Eclipse 3.0 does not support robust enough classpath management for plugins to make this possible without major gyrations - this is the same issue that the Hibernate folks faced with their tooling.
  • Update By Example Method (???)
  • Optional support for JDK 5.0 generics on the appropriate methods (will be accomplished in the Eclipse 3.1 version)
  • Better "by example" functionality - could be improved if the iBATIS iterate tag were more functional.
  • Improve/complete the JavaDoc!

Your suggestions are welcome too!
